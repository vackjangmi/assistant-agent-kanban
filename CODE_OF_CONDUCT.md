# Code of Conduct

## 우리의 약속

Assistant Agent Kanban 커뮤니티는 모두가 안전하고 존중받는 환경에서 협업할 수 있기를 바랍니다. 프로젝트에 참여하는 모든 사람은 서로를 배려하고, 기술적 의견 차이가 있더라도 공격적이거나 배제적인 태도를 취하지 않기를 기대합니다.

우리는 다음과 같은 커뮤니티를 지향합니다.

- 서로에게 친절하고 존중하는 커뮤니티
- 초보자도 질문하기 쉬운 커뮤니티
- 피드백은 솔직하지만 공격적이지 않은 커뮤니티
- 기술적 논의와 사람에 대한 평가를 분리하는 커뮤니티

## 기대하는 행동

- 상대를 존중하는 언어를 사용합니다.
- 비판보다 맥락과 근거를 함께 제시합니다.
- 다른 사람의 시간과 노력을 존중합니다.
- 실수나 오해가 생기면 방어적 태도보다 문제 해결에 집중합니다.
- 리뷰, 이슈, 토론에서 구체적이고 재현 가능한 정보를 제공합니다.

## 허용되지 않는 행동

다음과 같은 행동은 이 커뮤니티에서 허용되지 않습니다.

- 모욕, 혐오 표현, 비하, 조롱
- 괴롭힘, 위협, 협박, 지속적인 공격적 언행
- 성별, 성적 지향, 장애, 외모, 인종, 국적, 종교, 경력 수준 등을 이유로 한 차별
- 악의적인 도발, 고의적인 분쟁 유도, 반복적인 비생산적 논쟁
- 타인의 개인정보나 민감한 정보를 동의 없이 공개하는 행위
- 프로젝트 공간에서 부적절하거나 성적인 언행 또는 이미지 공유

## 적용 범위

이 행동 강령은 다음 공간에 적용됩니다.

- GitHub Issues
- Pull Requests
- Discussions
- 코드 리뷰
- 프로젝트와 관련된 커뮤니케이션 채널

## 커뮤니티 운영자의 역할

프로젝트 운영자는 건강한 협업 환경을 유지하기 위해 다음 조치를 취할 수 있습니다.

- 문제 행동에 대한 수정 요청
- 부적절한 댓글, 이슈, PR, 토론 내용의 편집 또는 삭제
- 경고
- 일시적 또는 영구적 참여 제한

운영자는 커뮤니티의 안전과 지속 가능성을 최우선으로 판단합니다.

## 신고 방법

행동 강령 위반이 의심되는 상황을 발견하면 공개적인 충돌로 키우기보다, 가능한 한 비공개로 알려 주세요.

현재는 GitHub issue를 통한 공개 신고보다 저장소 관리자에게 직접 연락하거나, 적절한 private 채널을 사용하는 방식을 권장합니다. 공개 issue에 민감한 내용을 그대로 남기지 말아 주세요.

신고 시 포함하면 좋은 정보:

- 발생한 위치
- 관련 링크
- 발생 시각
- 상황 설명
- 가능하다면 스크린샷 또는 로그

## 좋은 커뮤니티를 함께 만듭니다

오픈소스는 코드만으로 유지되지 않습니다. 질문하는 사람, 답하는 사람, 문서를 다듬는 사람, 버그를 재현해 주는 사람 모두가 프로젝트의 중요한 기여자입니다.

서로를 존중하는 태도가 좋은 소프트웨어를 만듭니다.
